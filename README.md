# Minesweeper-2026

Классическая игра «Сапёр» с веб-интерфейсом, аккаунтами игроков, таблицей лидеров
и сохранением прогресса. Бэкенд написан на C++ с использованием фреймворка
Drogon, фронтенд — на чистом JavaScript, данные хранятся в PostgreSQL.

Развёрнута на сайте: `minesweeper.site`

## Возможности

- Классический игровой процесс «Сапёра» с различными уровнями сложности (возможно будет также добавлен кастомный режим).
- Регистрация и вход по логину/паролю (JWT-авторизация) либо игра в гостевом
  режиме без аккаунта.
- Сохранение и восстановление текущей партии, история результатов, таблица
  лидеров.
- Настройки профиля: никнейм, аватар.

## Технологический стек

- **Бэкенд**: C++20, [Drogon](https://github.com/drogonframework/drogon)
  (HTTP-контроллеры, асинхронная работа с БД), [jwt-cpp](https://github.com/Thalhammer/jwt-cpp)
  + libsodium для аутентификации.
- **База данных**: PostgreSQL.
- **Фронтенд**: HTML/CSS/JavaScript.
- **Зависимости**: управляются через vcpkg(см. `vcpkg.json`).
- **Развёртывание**: Docker / Docker Compose, обратный прокси Caddy
  (автоматический HTTPS через Let's Encrypt + rate limiter).

## Структура проекта

```
src/
  api/      — HTTP-контроллеры (аутентификация, игра, пользователи, таблица лидеров)
  app/      — точка входа и конфигурация приложения Drogon
  db/       — клиент БД и репозитории (пользователи, игры, рекорды)
  game/     — игровая логика («Сапёр»: поле, клетки, сложность, сессия)
  session/  — сессии игроков (в т. ч. гостевые)
web/        — статический фронтенд (HTML/CSS/JS)
migrations/ — SQL-миграции схемы базы данных
tests/      — unit/e2e тесты (GoogleTest)
caddy/      — сборка кастомного образа Caddy (с плагином ограничения частоты запросов)
```

## API

Все эндпоинты находятся под префиксом `/api`:

| Метод | Путь                  | Назначение                          |
|-------|-----------------------|-------------------------------------|
| POST  | `/api/auth/register`  | Регистрация нового пользователя     |
| POST  | `/api/auth/login`     | Вход и получение JWT-токена         |
| POST  | `/api/game/start`     | Начать новую партию                 |
| POST  | `/api/game/move`      | Открыть клетку                      |
| POST  | `/api/game/flag`      | Поставить/снять флажок              |
| POST  | `/api/game/revert`    | Отменить последний неверный ход     |
| GET   | `/api/game/state`     | Получить текущее состояние партии   |
| POST  | `/api/game/save`      | Сохранить партию                    |
| POST  | `/api/game/resume`    | Восстановить сохранённую партию     |
| GET   | `/api/leaderboard`    | Получить таблицу лидеров            |
| GET   | `/api/users/me`       | Получить данные текущего профиля    |
| PATCH | `/api/users/me`       | Обновить данные профиля (ник, аватар)|

## Локальный запуск

### Требования

- CMake, C++20, [vcpkg](https://github.com/microsoft/vcpkg).
- PostgreSQL (можно поднять через Docker, см. `docker-compose.yml`).

### Шаги

1. Скопируйте `.env.example` в `.env` и задайте свои значения
   (пароль БД, секрет для подписи JWT).
2. Поднимите базу данных:
   ```bash
   docker compose up -d db
   ```
   SQL-миграции из `migrations/` применятся автоматически при первом старте.
3. Соберите проект:
   ```bash
   cmake -B build -DCMAKE_TOOLCHAIN_FILE=$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake
   cmake --build build --target minesweeper
   ```
4. Запустите приложение:
   ```bash
   ./build/minesweeper
   ```
5. Откройте `http://localhost:8080` в браузере.

### Тесты

Модульные тесты собираются и запускаются через CTest:
```bash
cmake --build build --target unit_tests
ctest --test-dir build --output-on-failure
```

## Развёртывание

Проект разворачивается в Docker через `docker-compose.yml`:

```bash
docker compose up -d --build
```

Состав сервисов:

- **`app`** — само приложение (собирается из корневого `Dockerfile` многоступенчатой
  сборкой: vcpkg компилирует зависимости из исходников, итоговый образ — лёгкий
  рантайм-контейнер).
- **`db`** — PostgreSQL с проверкой готовности (`healthcheck`) и автоматическим применением
  миграций из `migrations/`.
- **`caddy`** — обратный прокси перед приложением: автоматически получает и
  обновляет TLS-сертификаты Let's Encrypt (см. `Caddyfile`) и ограничивает
  частоту запросов от одного клиента (защита от примитивных DDoS/перебора).

Перед первым запуском заполните `.env` (обязательны `POSTGRES_PASSWORD` и
`JWT_SECRET`) и укажите свой домен в `Caddyfile` вместо `minesweeper.site`.
